# 分析發現 (Findings)

## 1. 結構性發現

### 400 與 450副主任副廠長.py 是互補分流，非重複
部分單位設有兩位副主任（督導1 / 督導2），對應的簽核人不同，因此流程拆成兩個關卡：
- **400**：負責 `second == 1`（副主任1督導）的人員
- **450**：負責 `second == 2`（副主任2督導）的人員

兩檔最關鍵的互斥條件：
```python
# 400：排除 second==2（交由 450 處理）
if second == 2:
    return False

# 450：排除 second!=2（非指定督導一律跳過）
if second != 2:
    return False
```
兩者合起來覆蓋全體副主任督導分流，設計為有意的互補，不是重複。

### 850人事副主任 與 900人事主任 邏輯完全相同
排序後兩個檔案一字不差，只有職稱不同。推測兩個關卡是為了串接順序而分拆，邏輯本身尚未差異化。

### 700人事承辦 與 750二廠人事 是鏡像分流
- 700 的工會假條件：`vaid==67 and **not** (second==3 or onlySecond==2 or secondDept==1)` → 非二廠才審
- 750 的工會假條件：`vaid==67 and (second==3 or onlySecond==2 or secondDept==1)` → 是二廠才審
- 兩者合起來覆蓋全體，無交集，設計一致。

### 700人事承辦 多一個重複的安全防護條件
```python
if second==3 or plevel==2:
    return False
# ...（中間有其他條件）
if second==3:          # ← 重複，second==3 已在上方處理過
    return False
```
在 `plevel==2` 判斷之後，`second==3` 已無法單獨作用的情況，上方已排除。保留作為防禦性設計。

## 2. 已修正的問題（400副主任副廠長.py）

### [已修正] vaid==67 頂層排除封鎖了下方的例外條件
原先頂部 `if vaid == 67: return False` 使得下方針對特定職級的 `return True` 永遠不會被執行。
**修正方式**：將頂層的 `if vaid == 67: return False` 整行註釋掉，讓會務假可以流到下方的職級判斷。

### [已修正] 缺少 vaid 165, 166 的對應處理
原先底部條件只寫 `vaid == 67`，缺少 165（非監理事會務假）與 166（非理監事會務假，團體協約44條）。
**修正方式**：改為 `vaid in (67, 165, 166)`，與 450 的處理方式對齊。

---

### line 67 的 comment 已過時
原本 comment 寫「這段似乎與最上方的 vaid==67 衝突」，但因為頂層排除已被註釋掉，衝突已不存在，該 comment 可清除。

## 4. 規則共同模式（排序後浮現）

以下關卡的 Pass Quick 第一條都是防疫/會務假強制進入：
- 800人事組長：`vaid in (67, 117, 118, 119, 120, 165, 166)`
- 850人事副主任：同上
- 900人事主任：同上
- 1000工務副總：`vaid in (117, 118, 119, 120)`（工會假改為排除）
- 1100業務副總：`vaid in (5, 67, 165, 166) and plevel!=3 and nlevel<900`

### 補休群組在所有檔案中一致
`(35, 36, 37, 38, 109, 110, 111)` 這七個假別始終作為一組出現，從未拆分。

### 「組長申請公務(vaid==6)需審」出現在三個人事關卡
700人事承辦、750二廠人事、800人事組長 都有：
```python
if (...) and nlevel<=700 and vaid==6:
    return True
```
這是一條跨關卡的共同業務規則。

### 遞延休假一般人員豁免出現在四個高層關卡
`nlevel==100 and vaid==21 and hours<8` 在 400、1000、1100、1200 中作為排除條件。低層關卡（500、700、750、800）則未見此條件，推測低層本來就不審或已由其他條件排除。

## 5. 設計觀察

### plevel==6（業務企劃中心）在各關卡行為不一致
| 關卡 | 行為 |
|------|------|
| 700人事承辦 | 直接排除（return False） |
| 750二廠人事 | 有條件進入（特殊假別或>8HR） |
| 1000工務副總 | 有條件進入（>8HR 或連續>1天） |
| 1100業務副總 | 直接排除（return False） |

業務企劃中心的審核路徑依賴 750 和 1000，繞過 700 和 1100。

### 1200總經理 是結構最乾淨的關卡
排序後只有 2 條進入 + 2 條排除，邏輯邊界清晰，可作為其他關卡的參考範本。
