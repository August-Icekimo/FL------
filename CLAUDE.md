# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 專案說明

這是公司請假單系統的工作流引擎**關卡判定腳本集**。每支 `.py` 腳本對應一個簽核關卡，由主系統動態執行，判斷請假單是否需要經過該關卡。

**腳本不是獨立程式**：沒有 `import`、沒有 `main()`、沒有類別定義。主系統在執行時將請假單屬性注入為全域變數，腳本邏輯讀取這些變數並 `return True`（需審核）或 `return False`（跳過）。

## 檔案命名規則

`[數字][職稱].py` — 數字代表關卡職等順序（數字越大職位越高），例如：
- `500單位主官.py` → 股長以上單位主官
- `800人事組長.py` → 人事組長
- `1200總經理.py` → 總經理

## 核心變數（必須對照 `參數說明.md`）

| 變數 | 說明 |
|------|------|
| `vaid` | 假別代碼（如 `12`=事假、`20`=休假、`67`=理監事會務假） |
| `nlevel` | 申請人職級（`100`=一般、`500`=股長、`700`=組長、`750`=副主管、`800`=主管、`900`=副總、`1000`=總經理） |
| `plevel` | 簽核部門（`1`=一廠、`2`=二廠、`3`=工務副總督導、`4`=業務副總督導、`6`=業務企劃中心） |
| `hours` | 請假時數 |
| `continueDays` | 連續請假天數 |
| `master` | 人員群組主分類（`1`=廠長室、`2`=秘書室、`4`=總經理室） |
| `second` | 人員群組副分類（`3`=二廠人事業務範圍） |
| `secondDept` | 第二部門細分類（`3`=製證中心資通機電、`4`/`5`=製證品檢封裝股） |
| `onlySecond` | 特定副督導代碼（`2`=派駐二廠） |
| `linkuncheng` | 政風流程旗標（`1`=是） |

## 腳本撰寫慣例

**判斷順序**：先處理強制進入（`return True`），再處理強制跳過（`return False`），最後給預設回傳。

**補休假別群組**：常用補休代碼清單為 `(35, 36, 37, 38, 109, 110, 111)`，修改時須確認是否需要涵蓋此群組。

**會務假別**：`vaid in (67, 165, 166)` 代表各類工會/理監事會務假，三個代碼通常需一起處理。

**防疫假別**：`vaid in (117, 118, 119, 120)` 為防疫相關假別，通常在高層關卡強制進入。

**中文註解**：條件變更時加入中文說明，標注對應的行政規定；修改者可標注姓名（如 `#彬鴻修改`）以利追蹤。

**二廠分流邏輯**：`second==3`、`onlySecond==2`、`secondDept==1`、`plevel==2` 是判斷二廠相關人員的常見條件，二廠請假單通常有獨立的審核路徑。

## 修改關卡的流程

1. 先查 `參數說明.md` 確認涉及的 `vaid`、`nlevel`、`plevel` 代碼
2. 修改腳本時確保不破壞「強制進入」條件置於「強制跳過」之前的順序
3. 透過主系統測試環境模擬對應參數，確認流程如預期進入或跳過該關卡
