這是一份針對 FloWing 簽核引擎系統所編寫的 **「修改與關閉簽核關卡」標準作業程序 (SOP)**。

本文件已將所有查詢與修改步驟「變數宣告化 (`DECLARE`)」，並修正了原始語法中遺漏的逗號（`SET fst_isValid = '0', fst_stime = ''`），方便您直接複製、修改變數值後安全執行。

# 📝 SOP：FloWing 簽核關卡修改與關閉維護作業

## 📌 作業目的

當使用者反映請假單因特殊原因卡單、誤按同意、需要跳過特定審核人，或是需要將單據人工導向正確的審核層級（如副總關卡）時，透過後端資料庫介入修正流程實例狀態。

## ⚠️ 事前注意事項

1. **資料備份建議**：執行任何 `UPDATE` 前，請務必確認 `WHERE` 條件的變數範圍完全正確。
2. **單號特性**：使用者前台 UI 看到的單號通常與底層的 `flow_id` 不同，必須遵循本 SOP 的「定位階段」逐步查找。

---

## 🔍 第一階段：定位與調查（尋找目標單據與關卡）

### 步驟一：查找申請人的內部使用者代碼 (`uid`)

由於前台只提供工號或姓名，必須先至組織表撈出系統底層識別用的 `uid`。

```sql
-- ==========================================
-- 步驟一：透過關鍵字（如工號或姓名）查找 uid
-- ==========================================
DECLARE @SearchKey NVARCHAR(50) = '%2286%'; -- 🔍 請在此輸入要查詢的工號或姓名關鍵字

SELECT uid, name
FROM eip..people_ldap
WHERE name LIKE @SearchKey 

```

---

### 步驟二：結合時間與 `uid` 盲查鎖定流程 ID (`flow_id`)

依據使用者提供的送單時間區間，以及上一步取得的 `uid`，肉眼確認並鎖定目標 `flow_id`。

```sql
-- ==========================================
-- 步驟二：透過時間範圍與 uid 鎖定 flow_id
-- ==========================================
DECLARE @TargetUid INT = 1115;                        -- 🔍 請輸入步驟一查到的 uid
DECLARE @StartTime DATETIME = '2026-05-29 09:25:00'; -- 🔍 請輸入送單時間點之前的時間

SELECT TOP 100 
*
FROM edoc..flow_instance
WHERE flow_createtime >= @StartTime
  AND flow_uid = @TargetUid
ORDER BY flow_createtime DESC;

```

---

### 步驟三：檢視該單據展開的簽核流程歷程

拿到目標 `flow_id` 後，至狀態表檢視目前已經產生的所有簽核關卡，從中抓出需要關閉或修改的關卡流水號（`fst_serial`）。

```sql
-- ==========================================
-- 步驟三：檢視指定單據的所有簽核關卡明細
-- ==========================================
DECLARE @TargetFlowId INT = 645037; -- 🔍 請輸入步驟二鎖定的 flow_id

SELECT 
*
FROM edoc..flow_state
WHERE fst_did = CAST(@TargetFlowId AS VARCHAR(50))
ORDER BY fst_serial ASC;

```

---

## 🛠️ 第二階段：變更與修正（關閉關卡與調整指標）

### 步驟四：關閉特定簽核關卡（單一關卡）

將指定關卡設為失效（`fst_isValid = '0'`）並清空審核時間，使其不再干擾流程。

```sql
-- ==========================================
-- 步驟四：關閉單一指定的簽核關卡
-- ==========================================
DECLARE @TargetFlowId INT  = 645037;   -- ⚙️ 目標 flow_id (fst_did)
DECLARE @TargetSerial INT = 12531340; -- ⚙️ 要關閉的關卡流水號 (fst_serial)

UPDATE edoc..flow_state
SET fst_isValid = '0',
    fst_stime = NULL
WHERE fst_serial = @TargetSerial
  AND fst_did = CAST(@TargetFlowId AS VARCHAR(50));

-- 驗證更新結果
SELECT *
FROM edoc..flow_state 
WHERE fst_serial = @TargetSerial;

```

---

### 步驟五：調整流程當前審核指標（跳關/抽單）

關閉關卡後，必須修正流程主表的當前指標（`flow_instord`），前台待辦清單才會移動到新的正確審核人身上。

```sql
-- ==========================================
-- 步驟五：調整流程主表，導向正確的審核關卡節點
-- ==========================================
DECLARE @TargetFlowId INT        = 645037; -- ⚙️ 目標 flow_id
DECLARE @NewInstOrd NVARCHAR(10) = '1100'; -- ⚙️ 目標關卡代碼 (例如: '1100' 代表副總)

UPDATE edoc..flow_instance
SET flow_instord = @NewInstOrd
WHERE flow_id = @TargetFlowId;

-- 驗證更新結果
SELECT flow_id, flow_instord, flow_status 
FROM edoc..flow_instance 
WHERE flow_id = @TargetFlowId;

```

---

### 💡 進階備用步驟：批次關閉多個簽核關卡

若該單據後面有多個關卡需要一次廢棄（例如表單直接抽回、終止或大量跳關），可使用此批次腳本：

```sql
-- ==========================================
-- 備用步驟：一次批次關閉多個簽核關卡
-- ==========================================
DECLARE @TargetFlowId INT = 645037; -- ⚙️ 目標 flow_id (fst_did)

-- 執行批次關閉
UPDATE edoc..flow_state
SET fst_isValid = '0'
WHERE fst_did = CAST(@TargetFlowId AS VARCHAR(50))
  AND fst_serial IN (12531341, 12531343, 12531344, 12531346); -- ⚙️ 請在此填入所有欲關閉的流水號

-- 批次驗證結果
SELECT *
FROM edoc..flow_state 
WHERE fst_did = CAST(@TargetFlowId AS VARCHAR(50))
ORDER BY fst_serial ASC;

```
